## Design Context

### Users
招聘方、技术负责人、潜在合作者，评估毛际可作为高级前端工程师的技术深度与职业素养。访问场景：工作时间桌面端浏览为主，移动端碎片化浏览为辅。核心任务：3 秒内判断「这个人靠不靠谱」，然后深入了解项目经历与技术栈。

### Brand Personality
专业 · 沉稳 · 精致（Professional · Confident · Refined）

界面应传递信任感与能力感——一个能交付生产级代码的工程师。语气冷静自信，不浮夸，不自我推销。技术感通过细节体现（精准的间距、一致的 token、克制的动效），而非通过视觉噪音堆砌。

### Aesthetic Direction
现代极简 + 技术精准感。Indigo 主色配 Slate 中性色，Amber 作为次要强调。导航栏滚动后毛玻璃效果是唯一的 Glassmorphism 用法——有目的，不泛滥。Hero 区的 Matrix Rain 动效是个性化的技术人格表达，保留并强化。

布局左对齐为主，非对称构图，大量留白。避免：Bootstrap 风格的通用作品集、过度动画的"创意"风格、深色背景+霓虹光晕的 AI 美学、千篇一律的卡片网格。

参考气质：Linear.app 的精准克制、Vercel 的工程师美学、Stripe 的专业信任感。

### Design Principles
1. **内容优先** — 排版与间距服务于可读性，视觉装饰不喧宾夺主
2. **克制即自信** — 主色在一屏内出现不超过 3 次，留白是设计语言的一部分
3. **技术精准** — 严格遵循 8pt 间距系统与 Type Scale，无随意数值
4. **渐进披露** — 首屏简洁，交互中展现深度（hover 状态、滚动动画、AI 对话入口）
5. **双模式一等公民** — Light 与 Dark 模式均为精心设计，非简单色值反转

### Technical Context
- 框架：Vue 3 + TypeScript + Vite，无 UI 框架，纯自定义 CSS
- 设计 Token：已在 `frontend/css/variables.css` 定义，基于 CSS 自定义属性
- 主色：Indigo 600 (`#4F46E5`)，暗色下提亮至 Indigo 400 (`#818CF8`)
- 字体：Inter（西文）+ PingFang SC / Microsoft YaHei（中文）+ JetBrains Mono（代码）
- 断点：640 / 768 / 1024 / 1280 / 1536px，Mobile First
- 详细规范见 `design/DESIGN_SPEC.md`
